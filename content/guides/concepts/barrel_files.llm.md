# AdonisJS Barrel Files - LLM Reference

> Optimized for AI coding agents. For human-readable docs, see the full guide.

## Overview

Barrel files are auto-generated collections of exports that consolidate imports for controllers, events, and event listeners. Generated in `.adonisjs/server/`, they reduce import clutter in routes files.

**Key principle**: Purely aesthetic/ergonomic - no performance impact. Completely optional.

**Use cases**: Applications with many controllers where routes files accumulate dozens of imports.

## Basic Usage

### Without Barrel Files

```ts
// start/routes.ts
import router from '@adonisjs/core/services/router'

const NewAccountController = () => import('#controllers/new_account_controller')
const SessionController = () => import('#controllers/session_controller')
const PostsController = () => import('#controllers/posts_controller')
const PostCommentsController = () => import('#controllers/post_comments_controller')

router.get('signup', [NewAccountController, 'create'])
router.get('posts', [PostsController, 'index'])
```

### With Barrel Files (Default)

```ts
// start/routes.ts
import router from '@adonisjs/core/services/router'
import { controllers } from '#generated/controllers'

router.get('signup', [controllers.NewAccount, 'create'])
router.get('posts', [controllers.Posts, 'index'])
```

## How Barrel Files Work

Auto-generated by dev server in `.adonisjs/server/`:

```ts
// .adonisjs/server/controllers.ts
export const controllers = {
  NewAccount: () => import('#controllers/new_account_controller'),
  Session: () => import('#controllers/session_controller'),
  Posts: () => import('#controllers/posts_controller'),
  PostComments: () => import('#controllers/post_comments_controller')
}
```

**Auto-generation**: Updates automatically when you create/delete files during `node ace serve`.

## File Locations

```
.adonisjs/server/
  controllers.ts   # Controller barrel file
  events.ts        # Event barrel file
  listeners.ts     # Event listener barrel file
```

Import aliases:
```ts
import { controllers } from '#generated/controllers'
import { events } from '#generated/events'
import { listeners } from '#generated/listeners'
```

## Performance

Zero performance impact - uses lazy loading:

```ts
{
  Posts: () => import('#controllers/posts_controller')  // Only loads when used
}
```

Controllers load on-demand, identical to direct imports.

## Configuration

### Disable Barrel Files

```ts
// adonisrc.ts
import { defineConfig } from '@adonisjs/core/app'

export default defineConfig({
  hooks: {
    init: [
      () => import('@adonisjs/core/build/plugins/index_entities'),
      {
        indexEntities: {
          controllers: false,   // Disable controller barrel
          events: true,         // Keep events barrel
          listeners: true       // Keep listeners barrel
        }
      }
    ]
  }
})
```

After disabling, manually remove files from `.adonisjs/server/` and update imports.

## Common Patterns

### Pattern: Consistent Barrel Usage

```ts
// start/routes.ts
import { controllers } from '#generated/controllers'

router.get('/posts', [controllers.Posts, 'index'])
router.get('/users', [controllers.Users, 'index'])
router.get('/comments', [controllers.Comments, 'index'])
```

### Pattern: Direct Imports

```ts
// start/routes.ts
const PostsController = () => import('#controllers/posts_controller')
const UsersController = () => import('#controllers/users_controller')

router.get('/posts', [PostsController, 'index'])
router.get('/users', [UsersController, 'index'])
```

## IDE Auto-Import Pitfall

IDEs may auto-import directly even when using barrels:

```ts
// Wrong - IDE added this
const PostsController = () => import('#controllers/posts_controller')
import { controllers } from '#generated/controllers'

router.get('posts', [PostsController, 'index'])    // Direct import
router.get('users', [controllers.Users, 'index'])  // Barrel import
```

**Fix**: Delete auto-import, use barrel namespace:

```ts
// Correct
import { controllers } from '#generated/controllers'

router.get('posts', [controllers.Posts, 'index'])
router.get('users', [controllers.Users, 'index'])
```

**Tip**: Type `controllers.` first to trigger autocomplete and prevent IDE auto-importing.

## Quick Reference

### Generated Files
- Location: `.adonisjs/server/`
- Types: `controllers.ts`, `events.ts`, `listeners.ts`
- Auto-updates during dev server
- DO NOT manually edit

### Import Aliases
- `#generated/controllers`
- `#generated/events`
- `#generated/listeners`

### Naming Convention
Controller file: `app/controllers/post_comments_controller.ts`
Barrel name: `controllers.PostComments`

## Important Notes

- Barrel files are optional - both approaches work identically
- No performance difference - uses lazy imports
- Auto-generated during `node ace serve`
- Purely aesthetic benefit (reduced import clutter)
- Can mix barrel and direct imports (not recommended for consistency)
- DO NOT edit `.adonisjs/` files - they're regenerated automatically
